We will go through some steps to illustrate how the edition of lazy variables is handled depending on their laziness level.


First, we will define two lazy variables (with the laziness asign operator ':-'), where each variable evaluates to a floating point number and its corresponding value is a uniform number between 0 and 1.

The '::' operator infers the data type of a variable depending on its value. In this case. the data type of x and y is lazy double.

  • :: x :- uniform();
  • :: y :- uniform();

Now, let's define three variables that depend on x and y:

  • array[] => array[] => lazy double s1 := [['x', 'y'], ['x - y', 'x + y']];
  • array[] => lazy array[] => double s2 := ['[ x, y ]', '[x - y, x + y]'];
  • lazy array[] => array[] => double s3 := '[[ x, y ], [x - y, x + y]]';

Note that the three variables defined above use the apostrophe to set laziness. This way it is not necessary to use the laziness assign operator.


Let's see the layout of these variables:

As you may note, although s1, s2 and s3 have different definitions, they are represented the same way in the UI (because of the analysis of structure of the value of each variable).
